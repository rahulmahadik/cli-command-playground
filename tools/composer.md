# Composer Commands Cheatsheet

> Composer is a dependency manager for PHP.

## Quick Reference

> **Preview** — This is a curated selection of key commands. Explore the full command list with examples, practice exercises, and more on [Command Playground](https://technoscripts.com/command-playground/?utm_source=github&utm_medium=repository&utm_campaign=explore-all).

| Command | Description |
|---------|-------------|
| `composer install` | Install all dependencies from lock file |
| `composer require` | Add and install a new package |
| `composer require --dev` | Add a development dependency |
| `composer create-project` | Create project from a package |
| `composer remove` | Remove a package dependency |
| `composer update` | Update dependencies to latest versions |
| `composer self-update` | Update Composer itself to latest |
| `composer outdated` | Show outdated package dependencies |
| `composer show` | Display installed package information |
| `composer search` | Search for packages on Packagist |
| `composer depends` | Show which packages depend on another |
| `composer why` | Show why a package is installed |
| `composer init` | Initialize a new composer.json file |
| `composer validate` | Validate composer.json structure |
| `composer dump-autoload` | Regenerate the autoloader files |
| `composer run-script` | Run a defined Composer script |
| `composer config` | Set or get configuration values |
| `composer clear-cache` | Clear the Composer package cache |
| `composer global require` | Install a package globally |
| `composer diagnose` | Diagnose common Composer issues |
| `composer audit` | Check for known security vulnerabilities |
| `composer check-platform-reqs` | Verify platform requirements are met |
