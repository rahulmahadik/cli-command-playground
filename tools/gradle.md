# Gradle Commands Cheatsheet

> Gradle is a flexible, multi-language build system. Build Android, Java, and polyglot projects.

## Quick Reference

> **Preview** — This is a curated selection of key commands. Explore the full command list with examples, practice exercises, and more on [Command Playground](https://technoscripts.com/command-playground/?utm_source=github&utm_medium=repository&utm_campaign=explore-all).

| Command | Description |
|---------|-------------|
| `gradle build` | Build the project and run all checks |
| `gradle clean` | Delete the build output directory |
| `gradle assemble` | Assemble project outputs without testing |
| `gradle check` | Run all verification tasks |
| `gradle jar` | Create a JAR archive |
| `gradle bootJar` | Create a Spring Boot JAR |
| `gradle bootRun` | Run a Spring Boot application |
| `gradle classes` | Compile the main source classes |
| `gradle init` | Initialize a new Gradle project |
| `gradle wrapper` | Generate the Gradle wrapper files |
| `gradlew --version` | Display the Gradle wrapper version |
| `gradle tasks` | List all available tasks |
| `gradle tasks --all` | List all tasks including sub-tasks |
| `gradle help --task` | Show help for a specific task |
| `gradle -x` | Exclude a task from execution |
| `gradle --rerun-tasks` | Force re-execution of all tasks |
| `gradle --continue` | Continue build after a task failure |
| `gradle dependencies` | Display the dependency tree |
| `gradle dependencyInsight` | Show insight into a specific dependency |
| `gradle --refresh-dependencies` | Refresh all cached dependencies |
| `gradle buildEnvironment` | Display buildscript dependencies |
| `gradle outgoingVariants` | Display outgoing variants |
| `gradle test` | Run all unit tests |
| `gradle test --tests` | Run specific test classes or methods |
| `gradle test --debug-jvm` | Run tests with remote debugging |
| `gradle integrationTest` | Run integration tests |
| `gradle jacocoTestReport` | Generate code coverage report |
| `gradle properties` | Display all project properties |
| `gradle projects` | Display the sub-project hierarchy |
| `gradle model` | Display the project model |
| `gradle components` | Display project components |
| `gradle :subproject:task` | Run a task in a specific subproject |
| `gradle -p` | Use a specific project directory |
| `gradle --include-build` | Include another build |
| `gradle buildDependents` | Build and test dependent projects |
| `gradle buildNeeded` | Build and test required projects |
| `gradle --parallel` | Execute tasks in parallel |
| `gradle --build-cache` | Enable the build cache |
| `gradle --configuration-cache` | Enable the configuration cache |
| `gradle --max-workers` | Set maximum parallel workers |
| `gradle --daemon` | Run with the Gradle daemon |
| `gradle --no-daemon` | Run without the Gradle daemon |
| `gradle --stop` | Stop the Gradle daemon |
| `gradle --offline` | Execute without network access |
| `gradle --info` | Set log level to info |
| `gradle --debug` | Set log level to debug |
| `gradle --warn` | Set log level to warnings only |
| `gradle --quiet` | Suppress most output |
| `gradle --stacktrace` | Print full stacktrace on errors |
| `gradle --scan` | Publish a build scan report |
| `gradle --profile` | Generate a build performance report |
| `gradle --console` | Set the console output type |
| `gradle publish` | Publish artifacts to a repository |
| `gradle publishToMavenLocal` | Publish to the local Maven repository |
| `gradle bootBuildImage` | Build an OCI image for Spring Boot |
| `gradle uploadArchives` | Upload archives to a repository |
