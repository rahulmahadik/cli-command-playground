# .NET CLI Commands Cheatsheet

> .NET CLI is Microsoft's cross-platform development tool for .NET. Build, run, and publish apps.

## Quick Reference

> **Preview** — This is a curated selection of key commands. Explore the full command list with examples, practice exercises, and more on [Command Playground](https://technoscripts.com/command-playground/?utm_source=github&utm_medium=repository&utm_campaign=explore-all).

| Command | Description |
|---------|-------------|
| `dotnet new` | Create a new .NET project or file |
| `dotnet new console` | Create a new console application |
| `dotnet new webapi` | Create a new Web API project |
| `dotnet new mvc` | Create a new MVC web application |
| `dotnet sln` | Manage solution files |
| `dotnet restore` | Restore project dependencies |
| `dotnet add reference` | Add a project-to-project reference |
| `dotnet build` | Build a .NET project |
| `dotnet run` | Run a .NET project |
| `dotnet clean` | Clean build output artifacts |
| `dotnet watch` | Watch for changes and restart app |
| `dotnet test` | Run unit tests |
| `dotnet test --logger` | Run tests with a specific logger |
| `dotnet test --collect` | Collect code coverage data |
| `dotnet publish` | Publish the app for deployment |
| `dotnet publish -c Release` | Publish in Release configuration |
| `dotnet publish --self-contained` | Publish with runtime included |
| `dotnet ef migrations add` | Create a new EF Core migration |
| `dotnet ef database update` | Apply EF Core migrations to database |
| `dotnet ef migrations remove` | Remove the last EF Core migration |
| `dotnet ef database drop` | Drop the database |
| `dotnet ef dbcontext scaffold` | Reverse-engineer a database |
| `dotnet ef migrations list` | List all EF Core migrations |
| `dotnet ef migrations script` | Generate a SQL migration script |
| `dotnet add package` | Add a NuGet package reference |
| `dotnet remove package` | Remove a NuGet package reference |
| `dotnet list package` | List installed NuGet packages |
| `dotnet nuget push` | Push a package to a NuGet feed |
| `dotnet pack` | Create a NuGet package |
| `dotnet tool install` | Install a .NET global or local tool |
| `dotnet format` | Format source code to match style |
| `dotnet user-secrets` | Manage development-time secrets |
| `dotnet --info` | Display .NET installation information |
