# Webpack Commands Cheatsheet

> Webpack is a JavaScript module bundler. Bundle JS, CSS, and assets for production.

## Quick Reference

> **Preview** — This is a curated selection of key commands. Explore the full command list with examples, practice exercises, and more on [Command Playground](https://technoscripts.com/command-playground/?utm_source=github&utm_medium=repository&utm_campaign=explore-all).

| Command | Description |
|---------|-------------|
| `webpack` | Run webpack with default configuration |
| `webpack build` | Build the project explicitly |
| `webpack --mode production` | Build with production optimizations |
| `webpack --mode development` | Build with development settings |
| `webpack serve` | Start the webpack dev server |
| `webpack serve --open` | Start dev server and open browser |
| `webpack serve --hot` | Enable hot module replacement |
| `webpack serve --port` | Set the dev server port |
| `webpack --config` | Use a specific configuration file |
| `webpack --config-name` | Use a named configuration |
| `webpack --env` | Pass environment variables to config |
| `webpack --optimization-minimize` | Enable output minification |
| `webpack --optimization-split-chunks` | Enable code splitting |
| `webpack --stats` | Control build statistics output |
| `webpack --json` | Output build stats as JSON |
| `webpack --profile` | Capture timing information for modules |
| `webpack --progress` | Show build progress percentage |
| `webpack --node-env` | Set the Node.js environment |
| `webpack --define-process-env-node-env` | Define process.env.NODE_ENV |
| `webpack --output-path` | Set the output directory path |
| `webpack --output-filename` | Set the output file name pattern |
| `webpack --watch` | Watch for file changes and rebuild |
| `webpack watch` | Start webpack in watch mode |
| `webpack init` | Initialize a webpack configuration |
| `webpack info` | Display webpack environment info |
| `webpack version` | Display the webpack version |
