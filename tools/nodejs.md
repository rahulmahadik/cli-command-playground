# Node.js Commands Cheatsheet

> Node.js is a JavaScript runtime built on Chrome's V8 engine. Run JavaScript on the server, build APIs.

## Quick Reference

> **Preview** — This is a curated selection of key commands. Explore the full command list with examples, practice exercises, and more on [Command Playground](https://technoscripts.com/command-playground/?utm_source=github&utm_medium=repository&utm_campaign=explore-all).

| Command | Description |
|---------|-------------|
| `node` | Start Node.js or execute a script |
| `node -e` | Evaluate JavaScript from command line |
| `node -p` | Evaluate and print an expression |
| `node --watch` | Watch for file changes and restart |
| `node --experimental-modules` | Enable ES module support |
| `node --run` | Run a script from package.json |
| `node --experimental-strip-types` | Run TypeScript files directly |
| `node (REPL)` | Start interactive REPL session |
| `.help` | Show REPL help commands |
| `.load` | Load a file into the REPL |
| `.save` | Save REPL session to a file |
| `node inspect` | Start the built-in debugger |
| `node --inspect` | Enable remote debugging on a port |
| `node --inspect-brk` | Debug and break on first line |
| `node -c` | Check script syntax without executing |
| `node --print require.resolve` | Print resolved module path |
| `node --experimental-import-meta-resolve` | Enable import.meta.resolve |
| `node --prof` | Generate V8 profiler output |
| `node --prof-process` | Process V8 profiler output |
| `node --cpu-prof` | Generate CPU profile on exit |
| `node --heap-prof` | Generate heap profile on exit |
| `node --env-file` | Load environment variables from file |
| `node --max-old-space-size` | Set maximum V8 heap memory size |
| `node --no-warnings` | Suppress all process warnings |
| `node --version` | Display the Node.js version |
| `node -v` | Display version (short form) |
| `node -p process.versions` | Print all dependency versions |
| `node --test` | Run built-in test runner |
| `node --test-reporter` | Set test result reporter format |
| `node --test-coverage` | Enable test code coverage reporting |
